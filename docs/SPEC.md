# SoliTCG エンジンビジネスルール仕様書

**Version**: 1.0
**Last Updated**: 2026-04-06
**Owner**: Architect Agent

> このドキュメントはエンジンが実装すべきルールの権威ある仕様書。
> プレイヤー向けの説明は `docs/GAME_RULES.md` を参照。
> ここに変更を加える場合は `docs/GAME_RULES.md` との整合を確認し、影響があれば ADR を作成すること。

---

## 1. ゾーン定義

| ゾーン | 定数名 | 上限 | 性質 |
|---|---|---|---|
| 手札 | `hand` | 制限なし | プレイヤー参照可能 |
| デッキ | `deck` | 40 枚（構築済み） | 非公開・順序あり |
| 場 | `board` | 制限なし | 表向き永続ゾーン |
| ドメイン | `domain` | 1 枚 | `set_domain` op が置換を管理 |
| 墓地 | `grave` | 制限なし | 順序あり（直近が末尾） |
| 除外 | `banish` | 制限なし | ゲームから取り除かれる |
| エクストラ | `extra` | 10 枚 | ritual / arcane / relic のみ |

---

## 2. ターン構成（エンジン処理順）

1. **ドローフェイズ**: デッキから 1 枚引く。
2. **メインフェイズ**: プレイヤーはカードを任意の順序でプレイできる。フェイズ内の行動回数に制限なし。
3. **エンドフェイズ**: 勝敗条件を評価し、未達の場合は相手ターンへ遷移する。

---

## 3. カードタイプ別ふるまいマトリクス

| type | 場に残るか | プレイ後に墓地へ | HP を持つか | ドメイン制限 | エクストラ編入可 |
|---|---|---|---|---|---|
| monster | yes | no | yes | no | no |
| ritual | yes | no | yes | no | yes |
| spell | no | yes（解決後） | no | no | no |
| arcane | no | yes（解決後） | no | no | yes |
| artifact | yes | no | no | no | no |
| relic | yes（artifact 互換） | no | no | no | yes |
| equip | yes（装着状態） | no | no | no | no |
| domain | yes（domain ゾーン） | no | no | max 1 | no |

---

## 4. ドメインカード置換裁定

新しいドメインカードをプレイした際、すでにドメインが存在する場合の処理順序：

1. 新カードの `on_play` トリガーをキューに追加。
2. 旧ドメインカードを `grave` へ移動。
3. 旧カードの `on_destroy` トリガーをキューに追加。
4. トリガーキューを FIFO で解決する。

---

## 5. トリガー解決ルール

- **解決順**: FIFO（First In, First Out）。プレイヤーはキューの順序を操作できない。
- **最大反復数**: `resolveAll()` 1 回の呼び出しで最大 100 回のトリガー処理（無限ループガード）。
- **`pre` 評価タイミング**: 解決時点（エンキュー時ではなく）に評価する。
- **`pre` 失敗時**: そのトリガーはスキップ（再キューしない）。後続トリガーの処理は継続する。

---

## 6. `pre` 条件式の制約

- フィールドや状態の**読み取りのみ**許可。
- `pre` 式内での状態変更（`draw`・`destroy` 等の op 呼び出し）は禁止。
- 評価失敗（構文エラー等）はスキップ扱い。

---

## 7. デッキ構築ルール

| ルール | 値 |
|---|---|
| メインデッキ枚数 | ちょうど 40 枚 |
| 同名カード（メイン） | 最大 4 枚 |
| エクストラデッキ枚数 | 最大 10 枚 |
| 同名カード（エクストラ） | 最大 1 枚 |
| エクストラ編入可能タイプ | ritual / arcane / relic |
| 同名カードのカウント | メインとエクストラは別カウント |

---

## 8. HP と破壊ルール

- `monster` / `ritual` のみ `hp` を持つ。
- `hp <= 0` になった瞬間に破壊される（エンドフェイズまで待たない）。
- 破壊時に `on_destroy` トリガーを発火する。
- `hp` の変動は `modify_stat` op でのみ発生する。

---

## 9. 勝敗条件

| 条件 | 判定 |
|---|---|
| ライフ勝利 | 相手のライフが 0 以下（初期値: 8000） |
| デッキ勝利 | エンドフェイズ時点で相手デッキが 0 枚 |
| 効果勝利 | `win_if` op の条件が満たされた瞬間 |
| 効果敗北 | `lose_if` op の条件が満たされた瞬間 |

> ドロー不能による即時敗北はない。ターン終了時に判定する。

---

## 10. 式評価システム（ExpressionEvaluator）

対応する参照子：

| 参照子 | 意味 |
|---|---|
| `hand.count` | 手札の枚数 |
| `deck.count` | デッキの残り枚数 |
| `grave.count` | 墓地の枚数 |
| `field.exists` | 場にカードが存在するか |
| `spells_cast_this_turn` | このターン詠唱したスペル数 |
| `count(type:'X', zone:'Y')` | 指定ゾーンの指定タイプのカード枚数 |

対応する演算子: `>= <= > < == != && \|\| !`
