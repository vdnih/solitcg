# ADR-004: FIFO トリガーキューによる自動解決

## Status
Accepted

## Date
2025-10-22

## Context

TCG における「チェーン」や「スタック」はプレイヤーに効果の割り込み順序を選択させる仕組みだが、これは複雑な UI とゲームロジックを必要とする。
ソリティア形式では相手ターンの割り込みは存在しないが、プレイヤー自身が発動タイミングを選択するチェーン形式にするかどうかは設計上の判断が必要。

## Decision

誘発（トリガー）は FIFO キューで自動解決する。プレイヤーはキューの順序を操作できない。

- 誘発が発生した順にキューへ追加（エンキュー）する。
- `resolveAll()` が呼び出されると、キューを先頭から順次処理する。
- 無限ループ防止のため、1回の `resolveAll()` で最大 100 回の処理制限を設ける。
- `pre` 条件は解決時点（エンキュー時ではなく）に評価する。

## Consequences

**Positive:**
- プレイヤーへの選択を排除し、「コンボが自動で連鎖する」体験を実現。
- ゲームロジックがシンプルになり、デバッグ・テストが容易。
- ソリティア形式（PRODUCT_VISION.md の「Solitaire Purity」）との整合性が高い。

**Negative:**
- プレイヤーが「効果の解決順を戦略的に選択する」楽しみは提供できない。
- 将来的に「選択型チェーン」を導入したい場合、キューシステムの根本的な変更が必要。
