# ソリティアTCG ルール仕様 v0.5.0

## 1. ゲーム概要

* 一人用ソリティア形式のTCG。実際の対戦相手は存在しない。
* ライフ・デッキ枚数などの「相手側の数値」は仮想的な目標値として存在し、UIは対戦カードゲーム風に表現される。
* 割り込み処理は存在せず、処理は自分のターン内で完結する。
* バトル（戦闘）は未導入。カード効果のみで勝敗や盤面進行が決まる。
* デッキを削る・リソースを循環させる・コンボを構築することが主目的。

---

## 2. 勝利条件

### 2.1 基本勝利条件

* **ライフ勝利**：仮想ライフ（初期8000）を0以下にする。
* **デッキ勝利**：ターン終了時、仮想デッキが0枚であれば勝利。
    * ※「次のターンにドローできない」ことが確定した時点で敗北扱いとする。
* **効果勝利**：カードの効果により明示的に勝利条件を満たす。

---

## 3. デッキ構築

* **メインデッキ**：ちょうど40枚、同名カードは最大4枚。
* **エクストラデッキ**：最大10枚、同名カードは最大1枚。
* エクストラデッキには以下を含められる：
    * 儀式モンスター（ritual）
    * 秘術（arcane）
    * レリック（relic）
* いずれも通常のモンスター・魔法・アーティファクトと同じカテゴリ扱い（サポートカードの対象になる）。
* 同名カードのカウントはメインとエクストラで別カウント。

---

## 4. ゲームの流れ

### 4.1 初期設定

1. プレイヤーはデッキをシャッフル。
2. 初期手札を5枚引く。

### 4.2 ターン構成

1. **ドロー**：1枚引く。
2. **メインフェイズ**：カードのプレイ、能力発動など自由に行う。
    * 場（board）には出せるカード数の上限なし。
    * フィールド全体に影響するカードは**ドメイン（domain）**として同時に1枚まで設置可。
3. **ターン終了判定**：
    * 仮想デッキが0枚 → 勝利。
    * 勝利条件が満たされていない場合、次のターンへ。

---

## 5. カードタイプ

| type | 説明 |
| :--- | :--- |
| monster | 通常モンスター。hp 必須。atk / def は効果参照用（任意）。 |
| ritual | 儀式モンスター。monster 同様。エクストラに編入可。 |
| spell | 通常魔法。プレイ後に墓地へ。 |
| arcane | 秘術。強力効果カード。プレイ後に墓地へ。エクストラに編入可。 |
| artifact | アーティファクト。場に残る永続カード。 |
| relic | レリック。artifact 互換。エクストラに編入可。 |
| equip | 装備カード。対象モンスターに装着した状態で場に残る。 |
| domain | フィールド全体に影響。同時に1枚まで。 |

---

## 6. カードスキーマ（概要）

```yaml
id: string          # 一意なID
name: string        # カード名
type: monster | ritual | spell | arcane | equip | artifact | relic | domain
tags: [string]      # 任意の分類タグ
text: string        # 説明文
version: integer    # カードデータバージョン

stats:              # monster / ritual のみ（hp 必須、atk / def は任意）
  atk: integer      # 攻撃力（効果参照用。0以上）
  def: integer      # 防御力（効果参照用。0以上）
  hp:  integer      # ヒットポイント（0以上。hp <= 0 で破壊）

abilities:
  - when: on_play | on_destroy | on_discard | activated
    pre: [ "条件式", ... ]   # 発動前提：状態チェックのみ
    effect:
      - { op: ..., ... }
```

詳細なカード定義仕様は `docs/CARD_YAML_SPEC.md` を参照。

---

## 7. 発動条件効果（pre と効果0）

* `pre`：フィールドや状態のチェックのみを行う。動作は禁止。
* **効果0**：`effect` の先頭に記載し、`pre` に対応したリソース消費を行う。
* 例：

```yaml
pre:
  - "count(type:'artifact', zone:'board:self') >= 2"
effect:
  - { op: destroy, target: "choose:self:artifact", count: 2 }  # 効果0（消費）
  - { op: draw, count: 3 }
```

---

## 8. HPと破壊ルール

* `monster` / `ritual` のみ `hp` を持つ。
* `hp <= 0` になった瞬間、そのカードは破壊される（エンドフェイズを待たない）。
* 破壊時には `on_destroy` 能力が誘発する。
* `hp` の変動は `modify_stat` op でのみ発生する。

---

## 9. フィールド構造

* **場（board）**：制限なくカードを出せる。
* **ドメイン（domain）**：場全体に影響するカード。常に1枚まで。
* **墓地（grave）**：破壊・使用済みカードが送られる。
* **エクストラ（extra）**：儀式・秘術・レリック専用の別山札。

---

## 10. トリガー解決ルール

* 解決順は **FIFO**（先入れ先出し）。
* 同一タイミングで複数のトリガーが発生した場合、**プレイヤーがキュー投入順を選択**できる。
* `resolveAll()` 1回の呼び出しで最大100回のトリガー処理（無限ループガード）。
* `pre` 条件はエンキュー時ではなく**解決時点**で評価する。

---

## 11. `activated` 能力の制限

* `activated` 能力は **1ターンに1度** のみ発動できる（エンジン強制）。
* 同一カードで複数の `activated` 能力を持つ場合も、1ターン合計1度まで。

---

## 12. 補足ルール

* 割り込み処理は存在しない。
* 手札枚数に上限はない。
* ドロー不能になっても即敗北せず、ターン終了時に判定する。
